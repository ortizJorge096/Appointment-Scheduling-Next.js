Accesibilidad (WCAG 2.1 AA)

Contraste: texto ink sobre beige/blanco supera 4.5:1; el dorado se usa como acento o sobre fondo oscuro, nunca como texto pequeño sobre claro. Foco visible con anillo dorado en todos los campos. Targets táctiles ≥44px. Estados no solo por color (los badges incluyen texto, no solo tono). Tipografía mínima 14px en móvil, jerarquía semántica h1→h4, y soporte de prefers-reduced-motion para las animaciones.

Implementación en Next.js + Tailwind

Tus tokens ya viven en tailwind.config.ts (gold, beige, ink, Cormorant + DM Sans). Recomendaciones para llevar este rediseño al repo:

  • Añadir Pinyon Script al logo vía next/font/google (evita CLS y FOUT).
  • Server Components para landing y galería (S3) + ISR; el flujo /agendar como Client Component con el estado de pasos en useReducer.
  • Disponibilidad real desde /api/availability con caché corto; deshabilitar slots ocupados en el render (cero callejones sin salida).
  • Reutilizar tus ui/Button, Card, Input, Badge, Modal con estas variantes y tokens.
  • Skeletons con loading.tsx por ruta; next/image para la galería con blur placeholder.
  • FAB de WhatsApp como componente fijo global; microinteracciones con las keyframes fade-up/fade-in ya definidas + respeto a motion-reduce.
